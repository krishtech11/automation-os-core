# 🤝 Contributing to UAOS

Thanks for your interest in contributing to **UAOS (Unified AI Automation Operating System)**.

This project is designed as a modular, distributed automation platform. Please follow the guidelines below to ensure consistency and maintainability.

---

## 🚀 Getting Started

1. **Fork the repository**

2. **Clone your fork**

   ```bash
   git clone https://github.com/your-username/UAOS.git
   cd UAOS
   ```

3. **Create a new branch**

   ```bash
   git checkout -b feature/your-feature-name
   ```

4. **Make your changes**

5. **Commit your changes**

   ```bash
   git commit -m "Add: short description of your feature"
   ```

6. **Push to your fork**

   ```bash
   git push origin feature/your-feature-name
   ```

7. **Open a Pull Request**

---

## 📌 Contribution Guidelines

* Follow the existing project structure
* Keep code modular and readable
* Avoid unnecessary dependencies
* Write meaningful commit messages
* Add comments where logic is complex
* Test your changes before submitting

---

## 🧩 Project Structure Rules

* Backend logic → `backend/app/`
* API routes → `backend/app/api/`
* Workflows → `backend/app/workflows/`
* Frontend components → `frontend/src/components/`

Do not introduce random folders or break the structure.

---

## ⚙️ Development Guidelines

* Use environment variables (no hardcoding secrets)
* Ensure database operations are safe and optimized
* Handle errors properly (no silent failures)
* Maintain compatibility with existing APIs

---

## 🧪 Testing

* Manually test your changes before submitting
* Ensure existing functionality is not broken
* Validate API responses and workflows

---

## 🔍 Pull Request Rules

* Clearly describe what your PR does
* Link related issues (if any)
* Keep PRs focused (avoid mixing multiple features)
* Large changes should be discussed before implementation

---

## 🚫 What NOT to Do

* Do not commit sensitive data (API keys, credentials)
* Do not push directly to `main`
* Do not introduce breaking changes without discussion
* Do not submit incomplete features

---

## 🛡️ Code Review

All contributions are reviewed before merging.
Feedback may be provided for improvements — please address them before approval.

---

## 📬 Need Help?

If you’re unsure about something, open an issue before starting.
